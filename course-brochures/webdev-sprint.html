About This Course
What is Build Your First Website — 3 Weeks?

This sprint is built around one goal: by the end of Week 3, you have a real, deployed personal portfolio website live on the internet with your own domain (or a free subdomain) that you built yourself from scratch. Not a template. Not a drag-and-drop builder. Actual HTML, CSS, and JavaScript — code you understand and can modify.


The sprint meets three evenings a week for three weeks — Monday, Wednesday, and Friday at 7 PM IST. Each 90-minute session teaches you what you need and immediately applies it to the project. Week 1 covers HTML structure and CSS styling — by Friday you have a styled personal bio page. Week 2 adds JavaScript — by Friday you have a dynamic page that fetches data and responds to user interaction. Week 3 assembles everything into a complete portfolio and deploys it live with GitHub and Vercel.


All sessions are recorded so you can rewatch anything you missed or found difficult. The format is deliberately conversational — small group, questions encouraged, code reviewed live. You build something real every single session — no toy exercises, no "hello world" that goes nowhere.

Is This Right for You?
Who Should Attend
✓ This is for you if…
  • You're a complete beginner who wants to build a real website — not a drag-and-drop template but actual code you own and understand
  • You're a student, freelancer, or professional who wants a portfolio website to showcase your work online
  • You can commit three evenings a week for three weeks — the sprint format requires showing up regularly
  • You want to understand the foundations before learning React, Next.js, or any modern framework
✗ This is NOT for you if…
  • You already know HTML, CSS, and JavaScript — see the React Foundations sprint for the next level

Prerequisites: None — absolute beginners are welcome. You need a laptop, a text editor (we recommend VS Code, free to install), and a stable internet connection.
Curriculum
Three Weeks, Six Sessions, One Real Project
Week 1
HTML & CSS
  • HTML document structure & semantic elements — how a webpage is structured; headings, paragraphs, links, images, lists, and semantic tags (header, main, section, footer) and why they matter for accessibility and SEO
  • CSS selectors, box model, typography — how CSS targets elements, the box model (margin, border, padding, content) as the foundation of every layout, and typography that actually looks good
  • Flexbox & Grid — responsive layouts — the two modern CSS layout systems that replaced years of float hacks; building layouts that work on any screen size
  • Your first page: personal bio — by end of Week 1 you have a styled, responsive personal bio page that looks professional and works on mobile
Week 2
JavaScript
  • Variables, functions, events — JavaScript fundamentals; how the language handles data, how to write reusable logic, and how to respond to user actions (clicks, typing, hovering)
  • DOM manipulation — changing the page live — using JavaScript to modify HTML and CSS dynamically; selecting elements, changing content and styles, adding and removing elements
  • Fetch API — pulling data from the internet — making HTTP requests in JavaScript; parsing JSON responses; displaying live data on your page (GitHub profile, weather, etc.)
  • Project: dynamic portfolio page — by end of Week 2 your portfolio page fetches your GitHub repositories and displays them dynamically, updating automatically without page refresh
Week 3
Build & Deploy
  • Project: complete personal portfolio site — adding a projects section, contact form, navigation, and polishing every detail; a site you'd actually share with potential employers or clients
  • GitHub — version control basics — creating a repository, committing changes, pushing to GitHub; the workflow every developer uses and the foundation for all future collaboration
  • Deploy live with Vercel (free tier) — connecting your GitHub repository to Vercel; your site goes live at a real URL in under 5 minutes; automatic re-deployment on every push
  • Code review & next steps — instructor reviews your deployed site and gives specific, actionable feedback; a clear next-steps plan for continuing into React or backend development
