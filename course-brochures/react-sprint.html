About This Course
What is React Foundations — 4 Weeks?

React powers the front end of Facebook, Airbnb, Netflix, Notion, and thousands of other modern applications. It's the dominant way to build web UIs today and the first framework most companies expect you to know. This sprint teaches React the right way — starting with the mental model that makes React's design decisions obvious, then building real projects that cement each concept.


The sprint meets three evenings a week — Tuesday, Thursday, and Saturday at 8 PM IST — for four weeks, totalling twelve live 90-minute sessions. Each week has a theme: the React mental model, data and effects, routing and forms, and production readiness. Each theme ends with a project that pulls together everything covered.


By the end of Week 4 you'll have built a live search UI, a multi-page dashboard, and a full CRUD application — all deployed to production. You'll understand state management, side effects, routing, custom hooks, and performance optimisation. More importantly, you'll have the mental model to debug React applications and learn new parts of the ecosystem independently.

Is This Right for You?
Who Should Attend
✓ This is for you if…
  • You know HTML, CSS, and basic JavaScript and want to learn React — the most in-demand frontend skill in the industry
  • You've tried learning React from documentation or YouTube and find it overwhelming without structure
  • You want to build dynamic, multi-page web applications that feel like real products rather than static pages
  • You're preparing for frontend developer roles and need React on your resume with real projects to show
✗ This is NOT for you if…
  • You don't know HTML, CSS, and JavaScript yet — complete the "Build Your First Website" sprint first
  • You're looking for Next.js, server-side rendering, or full-stack React — this sprint covers client-side React only
  • You already use React professionally — this is a foundations course, not an advanced patterns course
Prerequisites: HTML, CSS, and basic JavaScript — you should be comfortable with variables, functions, events, and DOM manipulation. No React experience needed.
Curriculum
Four Weeks, Twelve Sessions, Three Projects
Week 1
The React Mental Model
  • Why React — components, JSX, the virtual DOM — what problem React solves, the component model as the fundamental abstraction, and how JSX turns HTML-like syntax into JavaScript function calls
  • Props & component composition — passing data into components; building a UI from small, reusable pieces; the difference between presentational and container components
  • State with useState — React's core state hook; how state triggers re-renders; the rules of state and why you can't mutate it directly
  • Event handling — onClick, onChange, onSubmit, and the React event system; building interactive components that respond to user input
Week 2
Data & Effects
  • useEffect — data fetching & side effects — when and how to run code outside React's render cycle; fetching data from APIs, subscriptions, and cleanup; the dependency array explained
  • Lifting state up — sharing state between sibling components by moving it to a common ancestor; the pattern behind most real component architectures
  • Context API — sharing state across many levels of the component tree without prop drilling; when to use Context vs. when it's overkill
  • Project 1: live search UI — a real-time search interface that fetches from an API as the user types, with debouncing, loading states, and error handling
Week 3
Routing & Forms
  • React Router — multi-page apps — client-side routing with React Router v6; defining routes, navigating between pages, URL parameters, and nested routes
  • Forms & controlled inputs — managing form state in React; controlled vs. uncontrolled inputs; validation patterns and form submission handling
  • Custom hooks — extracting and reusing stateful logic; building your own hooks (useLocalStorage, useFetch, useDebounce) to keep components clean
  • Project 2: multi-page dashboard — a multi-route dashboard application with navigation, data tables, and a detail view; using React Router and Context together
Week 4
Production Ready
  • State management patterns — when useState is not enough; useReducer for complex state logic; an introduction to what Zustand and Redux solve and when you need them
  • Performance — useMemo, useCallback — understanding React re-renders; optimising expensive computations and stable references; when to optimise and when not to
  • Project 3: full CRUD app — a complete Create/Read/Update/Delete application with local state management, form handling, optimistic updates, and error states
  • Deployment & portfolio — deploying your React app to Vercel; structuring your portfolio to show React projects; what to learn next (Next.js, TypeScript, testing)

Which version of React does this course use?
React 18 with functional components and hooks throughout. Class components are not covered — they're legacy and not used in new projects.
Will TypeScript be used?
No — the sprint uses JavaScript. TypeScript adds important benefits for larger projects but is a distraction when learning React's core concepts. We recommend adding TypeScript after completing this sprint.
What's the pathway after this sprint?
The natural next step is Next.js (React for production with server-side rendering and API routes), TypeScript, and a testing library (Vitest/Testing Library). We'll share a specific roadmap at the end of Week 4.
