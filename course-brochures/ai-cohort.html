About This Course
What is Build with AI — Weekend Cohort?

Most people who are excited about AI are still just using it — copying prompts, trying different tools, watching demos. This weekend flips that. By Sunday evening you'll have moved from consumer to builder: you'll have written real code that calls AI APIs, implemented a RAG (Retrieval-Augmented Generation) system, built an AI writing assistant with memory, and deployed your own AI micro-product.


The cohort is structured around projects, not lectures. You learn a concept, immediately build with it, then move to the next. Day 1 takes you from understanding how LLMs work through your first two projects. Day 2 covers advanced patterns — function calling, agents, embeddings — and ends with you building and deploying your own AI project concept.


This is a hands-on coding cohort. You'll need basic Python (variables, functions, loops) but no AI knowledge at all. The emphasis throughout is on patterns: the code patterns that appear in almost every AI application, so you can adapt them to any project you want to build next.

Curriculum
Two Days, Four Blocks, Three Projects
Day 1 — Morning
AI Foundations & APIs
  • How LLMs work (intuition) — a fast, visual explanation of transformers, tokens, and context windows; just enough to make sense of what you're building with
  • OpenAI & Anthropic API setup — accounts, API keys, rate limits, pricing; everything you need to start calling LLMs in code
  • Prompt engineering — zero-shot, few-shot, and chain-of-thought prompting with live examples of what works and what doesn't; system vs. user prompts
  • Structured output & JSON mode — getting predictable, machine-parseable output from LLMs; the foundation of every serious AI application
Day 1 — Afternoon
Projects 1 & 2
  • Project 1: AI Q&A over your own documents (RAG basics) — implement a simple Retrieval-Augmented Generation system; upload documents, embed them, and answer questions with citations
  • Project 2: AI writing assistant with memory — build a conversational AI that remembers context across turns; understanding conversation history and memory patterns
  • Token limits, costs & best practices — understanding context windows, how to manage token usage, and keeping API costs under control in real applications
  • Error handling & fallbacks — building AI applications that degrade gracefully; retry logic, fallback models, and handling rate limits
Day 2 — Morning
Advanced Patterns
  • Function calling & tool use — giving LLMs the ability to call your code; the pattern behind AI assistants that can take actions in the real world
  • Agents — multi-step reasoning — building AI systems that can plan, execute steps, and respond to intermediate results; the ReAct pattern explained
  • Embeddings & semantic search — vector representations of text; building search that understands meaning, not just keywords
  • Evaluating AI outputs — how to measure whether your AI application is actually working; LLM-as-judge, human eval, and automated metrics
Day 2 — Afternoon
Project 3 & Deployment
  • Project 3: your own AI micro-SaaS concept — you design and build a small AI application around your own use case; the instructor provides live technical support
  • Deploying AI apps (Vercel + API keys) — shipping your application to a live URL; environment variables, secret management, and the deployment checklist
  • Cost optimisation — caching, prompt compression, model selection, and batching strategies to keep running costs low
  • What to build next — personal roadmap — based on your Project 3, a concrete next-steps plan for taking your idea further
